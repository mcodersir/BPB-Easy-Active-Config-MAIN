## معرفی کلی

BPB Easy Active Config MAIN v9 برای کاربرانی طراحی شده که یک تجربه ساده و راهنما‌دار می‌خواهند تا:

1. **حساب Cloudflare بسازند** و یک Worker از نوع VLESS-over-WebSocket دیپلوی کنند
2. **endpointهای IP کلادفلر را اسکن کنند** و بهترین‌ها را از نظر کیفیت و تأخیر پیدا کنند
3. **کانفیگ‌های پروکسی را دریافت، تست و تولید کنند** که واقعاً کار می‌کنند
4. **بهترین کانفیگ را خروجی بگیرند** و آماده استفاده در هر کلاینت سازگار باشند

این برنامه کاملاً به‌صورت محلی روی سیستم کاربر اجرا می‌شود و از یک سرور HTTP داخلی استفاده می‌کند. رابط کاربری کاملاً فارسی، RTL، ریسپانسیو و بدون نیاز به CDN خارجی است — تمام CSS، JS و آیکون‌ها داخل پروژه قرار دارند.

### تفاوت‌های نسخه v9 با نسخه‌های قبلی

نسخه 9 بهبودهای مهمی نسبت به نسخه‌های قبلی دارد:

- **پروب واقعی VLESS-over-WebSocket**: به‌جای تکیه صرف به تست TCP/TLS، نسخه v9 یک WebSocket Upgrade واقعی و سپس یک Probe سبک پروتکل VLESS انجام می‌دهد. این کار نتایجی بسیار نزدیک‌تر به کاربرد واقعی تولید می‌کند و خطای مثبت کاذب (وقتی endpoint به TCP جواب می‌دهد ولی واقعاً ترافیک VLESS را پروکسی نمی‌کند) را به‌شدت کاهش می‌دهد.

- **دستیار دیپلوی یکپارچه Cloudflare**: مرحله دیپلوی حالا کاملاً داخل Wizard ادغام شده. کاربران دیگر نیازی به آپلود دستی فایل Worker ندارند — برنامه از Cloudflare API با فرمت multipart/form-data برای آپلود مستقیم worker.js به اکانت خود کاربر استفاده می‌کند.

- **حالت هوشمند Auto**: موتور تست اول کانفیگ‌های اصلی BPB را امتحان می‌کند. فقط اگر هیچکدام جواب ندهد، به‌طور خودکار به تولید کانفیگ‌های اصلاح‌شده با Clean IP رفتار فال‌بک می‌کند و زمان کاربرانی که کانفیگ اصلی‌شان از قبل کار می‌کند هدر نمی‌رود.

- **بدون وابستگی خارجی**: بک‌اند Python فقط به کتابخانه استاندارد نیاز دارد — هیچ pip install لازم نیست. تمام JavaScript، CSS و کد Worker به‌صورت محلی باندل شده‌اند.

## راهنمای مرحله‌ای کاربر

### مرحله ۱: شروع و پیش‌نیازها

وقتی برای اولین بار برنامه را اجرا می‌کنی، صفحه خوش‌آمدگویی را می‌بینی. قبل از ادامه مطمئن شو:

- یک ایمیل قابل دسترسی داری (اگر تیمت ایمیل Atomic Mail داده، از همان استفاده کن)
- می‌دانی که این Worker و کانفیگ‌ها فقط برای اکانت و استفاده مجاز خودت است

### مرحله ۲: ایمیل و Cloudflare

1. اگر تیمت ایمیل Atomic Mail داده، دقیقاً همان ایمیل را استفاده کن
2. اگر ایمیل نداری، روی دکمه **Atomic Mail** بزن و یک ایمیل قابل دسترسی بساز
3. روی **ثبت‌نام Cloudflare** بزن تا حساب Cloudflare بسازی
4. از همان ایمیل برای Cloudflare و ایمیل پروژه استفاده کن
5. وقتی وارد Dashboard شدی، به مرحله بعد برو

### مرحله ۳: دیپلوی Worker

این مرحله فایل worker.js داخلی BPB را مستقیم روی حساب Cloudflare تو دیپلوی می‌کند:

1. **گرفتن API Token**: روی **API Tokens** بزن → My Profile → API Tokens → Create Token
2. دسترسی را تنظیم کن: **Account → Workers Scripts → Edit**
3. توکن را کپی کن و در فیلد **Cloudflare API Token** پیست کن
4. روی **«تست Token و پیدا کردن Account ID»** بزن — این کار به‌طور خودکار Account ID را پر می‌کند
5. یک UUID بساز: روی **«ساخت UUID امن»** بزن
6. روی **«Deploy فایل داخلی»** بزن تا Worker روی اکانتت آپلود شود

**خطای رایج**: اگر مسیر خطا شامل `accounts/cfat_...` بود، یعنی API Token را اشتباهی در فیلد Account ID گذاشتی. همیشه اول دکمه «تست Token» را بزن.

### مرحله ۴: اسکنر کیفیت IP (اختیاری)

این مرحله اختیاری است اما وقتی کانفیگ‌های اصلی خوب جواب نمی‌دهند توصیه می‌شود:

1. آدرس IP یا محدوده CIDR را وارد کن، یا از تولید تصادفی IP کلادفلر استفاده کن
2. پورت‌هایی که می‌خواهی تست شوند را انتخاب کن (443، 8443، 2053 و غیره)
3. روی **«اسکن کیفیت و ذخیره»** بزن تا endpointها تست شوند
4. بعد از اسکن، روی **«استفاده از IPهای سالم در تست کانفیگ»** بزن تا نتایج به مرحله بعد منتقل شود

اسکنر پروب‌های واقعی HTTP/TLS علیه endpointهای CDN کلادفلر انجام می‌دهد و بر اساس اتصال و تأخیر امتیازدهی می‌کند.

### مرحله ۵: Subscription و تست کانفیگ

1. **لینک Subscription را پیست کن** — این معمولاً URL Worker از مرحله 3 است (مثلاً `https://worker-name.account.workers.dev/sub`)
2. روی **«چک کردن لینک»** بزن تا مطمئن شوی Subscription قابل دسترسی است
3. یک حالت تست انتخاب کن:
   - **حالت هوشمند** (پیشنهادی): اول کانفیگ‌های اصلی BPB را تست می‌کند، اگر لازم شد به Clean IP برمی‌گردد
   - **فقط کانفیگ پایه**: فقط کانفیگ‌های Subscription اصلی را تست می‌کند
   - **فقط Clean IP**: فقط با IPهای اسکن‌شده/واردشده کانفیگ تولید می‌کند
4. روی **«شروع اجرا و ساخت خروجی قابل تست»** بزن

### مرحله ۶: خروجی نهایی

- بهترین کانفیگ به‌طور خودکار انتخاب و نمایش داده می‌شود
- روی **«کپی کانفیگ»** بزن تا در کلیپبورد کپی شود
- تمام کانفیگ‌های OK در `output/working_configs.txt` ذخیره می‌شوند
- بهترین کانفیگ در `output/best_active_config.txt` ذخیره می‌شود

## استفاده از خط فرمان (CLI)

برای کاربرانی که خط فرمان را ترجیح می‌دهند یا اتوماسیون نیاز دارند، CLI دو دستور اصلی دارد:

### اسکن IP

```bash
python3 cli.py scan-ips \
  --ips ips.txt \
  --cidrs cidrs.txt \
  --random 80 \
  --ports 443,8443,2053,2083,2087,2096 \
  --sni speed.cloudflare.com \
  --timeout 5 \
  --workers 48 \
  --limit 800
```

**پارامترها:**

| پارامتر | پیش‌فرض | توضیح |
|---------|---------|-------|
| `--ips` | "" | فایل شامل آدرس IP، هر خط یکی |
| `--cidrs` | "" | فایل شامل محدوده CIDR، هر خط یکی |
| `--random` | 80 | تعداد IP تصادفی کلادفلر تولیدشده |
| `--ports` | 443,8443,2053,2083,2087,2096 | پورت‌های تست (با کاما جدا) |
| `--sni` | speed.cloudflare.com | نام SNI برای پروب TLS |
| `--timeout` | 5 | تایم‌اوت اتصال به ثانیه |
| `--workers` | 48 | تعداد تست‌کننده همزمان |
| `--limit` | 800 | حداکثر تعداد endpoint برای اسکن |

### تست کانفیگ

```bash
python3 cli.py run \
  --sub "https://worker-name.account.workers.dev/sub" \
  --mode auto \
  --random 240 \
  --ports 443 \
  --timeout 6 \
  --workers 32 \
  --limit 1500
```

**پارامترها:**

| پارامتر | پیش‌فرض | توضیح |
|---------|---------|-------|
| `--sub` | (الزامی) | آدرس Subscription BPB |
| `--mode` | auto | حالت تست: auto، base، یا clean_ip |
| `--ips` | "" | فایل IP سالم برای حالت clean_ip |
| `--random` | 240 | IP تصادفی کلادفلر برای حالت auto/clean_ip |
| `--ports` | 443 | پورت‌های تولید کانفیگ (با کاما جدا) |
| `--timeout` | 6 | تایم‌اوت هر تست به ثانیه |
| `--workers` | 32 | تعداد تست‌کننده همزمان |
| `--limit` | 1500 | حداکثر تعداد کانفیگ برای تست |

## مرجع API

سرور HTTP محلی APIهای زیر را ارائه می‌دهد:

### GET /api/status

وضعیت برنامه و تنظیمات را برمی‌گرداند.

```json
{
  "app": "BPB Easy Active Config MAIN v9.0",
  "brand": "ساخته شده توسط mcoders",
  "no_cdn": true,
  "output_dir": "/path/to/output",
  "bundled_worker_present": true,
  "all_ports": [443, 8443, 2053, ...],
  "links": { ... }
}
```

### POST /api/fetch

آدرس Subscription را دریافت و پردازش می‌کند.

**بدنه درخواست:**
```json
{
  "subscription_url": "https://worker.workers.dev/sub",
  "timeout": 18
}
```

**پاسخ:**
```json
{
  "ok": true,
  "total_lines": 15,
  "supported_configs": 12,
  "examples": ["VLESS worker.workers.dev:443", ...],
  "saved": "/path/to/output/base_configs.txt"
}
```

### POST /api/scan-ips

endpointهای IP را از نظر کیفیت و تأخیر اسکن می‌کند.

**بدنه درخواست:**
```json
{
  "ip_text": "104.16.72.162\n172.64.100.8",
  "cidr_text": "104.16.0.0/24",
  "random_count": 160,
  "ip_limit": 900,
  "timeout": 4,
  "workers": 64,
  "sni_host": "speed.cloudflare.com",
  "ports": [443, 8443]
}
```

### POST /api/run

اندپوینت اجرای اصلی — Subscription را دریافت، کانفیگ تولید و تست می‌کند.

**بدنه درخواست:**
```json
{
  "subscription_url": "https://worker.workers.dev/sub",
  "mode": "auto",
  "timeout": 6,
  "workers": 32,
  "limit": 1600,
  "random_count": 240,
  "ip_list": "",
  "ports": [443, 8443]
}
```

### POST /api/cf-verify

توکن API کلادفلر را تأیید و اکانت‌های مرتبط را فهرست می‌کند.

**بدنه درخواست:**
```json
{
  "api_token": "your-cloudflare-api-token"
}
```

### POST /api/cf-deploy

فایل worker.js داخلی را روی کلادفلر دیپلوی می‌کند.

**بدنه درخواست:**
```json
{
  "api_token": "your-cloudflare-api-token",
  "account_id": "32-char-hex-account-id",
  "worker_name": "bpb-panel",
  "uuid": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "sub_path": "sub",
  "proxy_ip": ""
}
```

### POST /api/open-url

یک لینک امن و تأییدشده را در مرورگر پیش‌فرض باز می‌کند.

### GET /api/open-output

دایرکتوری خروجی را در فایل منیجر سیستم باز می‌کند.

### GET /api/open-integrated-folder

دایرکتوری سورس‌های ادغام‌شده را باز می‌کند.

## محافظت از کد

سورس‌کد این پروژه با چند لایه رمزگذاری در برابر کپی‌برداری و تغییر غیرمجاز محافظت شده است:

### فایل‌های Python
تمام فایل‌های سورس Python با روش **کامپایل بایت‌کد + فشرده‌سازی zlib + رمزگذاری base64** محافظت شده‌اند:
- سورس‌کد به آبجکت بایت‌کد Python کامپایل می‌شود
- بایت‌کد با zlib (سطح 9) فشرده می‌شود
- داده فشرده‌شده base64 رمزگذاری و در یک لودر جاسازی می‌شود
- سورس‌کد اصلی کاملاً از فایل‌ها حذف شده است
- در زمان اجرا، لودر رمزگشایی، باز فشرده‌سازی و بایت‌کد را اجرا می‌کند

این یعنی:
- نام متغیرها، کامنت‌ها و ساختار کد اصلی قابل بازیابی نیستند
- بازرسی مستقیم فایل فقط payload رمزگذاری‌شده را نشان می‌دهد
- کد همچنان به‌طور عادی اجرا می‌شود چون Python بایت‌کد را تفسیر می‌کند

### فایل‌های JavaScript
تمام فایل‌های JavaScript (app.js، worker.js) با **javascript-obfuscator** رمزگذاری شده‌اند:
- تسطیح جریان کنترل (Control flow flattening)
- تزریق کد مرده (Dead code injection)
- رمزگذاری آرایه رشته‌ای (base64)
- آستانه آرایه رشته‌ای 75%
- تغییر نام شناسه‌ها (نام‌های hexadecimal)
- تبدیل کلید آبجکت
- کد خودمحافظ (Self-defending)
- توالی‌های فرار یونیکد

### فایل‌های CSS
استایل‌شیت‌ها minify شده‌اند تا تمام فضای خالی، کامنت‌ها و قالب‌بندی خوانا حذف شود.

## مشکلات رایج و راه‌حل

### «پورت محلی آزاد پیدا نشد»
- برنامه‌های دیگری که پورت‌های 8765 تا 8824 را اشغال کرده‌اند ببندید
- یا تابع `find_free_port()` در `start.py` را برای استفاده از محدوده دیگری تغییر دهید

### «Account ID نامعتبر است» (cfat_ یا cfut_ در خطا)
- API Token را اشتباهی در فیلد Account ID گذاشتی
- روی «تست Token و پیدا کردن Account ID» بزن تا Account ID صحیح خودکار پر شود

### «workers.dev subdomain پیدا نشد»
- یک بار Cloudflare → Workers & Pages را باز کن
- اگر نام subdomain خواست، یک نام ساده انتخاب کن
- دوباره Deploy را بزن

### «Subscription خالی یا نامعتبر برمی‌گرداند»
- مطمئن شو Worker با موفقیت دیپلوی شده
- چک کن UUID بین دیپلوی و آدرس Subscription مطابقت داشته باشد
- آدرس Worker را مستقیم در مرورگر باز کن تا مطمئن شوی در حال اجراست

### «هیچ کانفیگ کاری پیدا نشد»
- اول اسکنر کیفیت IP را اجرا کن تا endpointهای خوب پیدا کنی
- به حالت Clean IP با IPهای اسکن‌شده برو
- تایم‌اوت و تعداد Worker را افزایش بده تا تست کامل‌تر شود

### خطای «Content-Type must be one of...» هنگام دیپلوی
- این نسخه از multipart/form-data برای آپلود Module Worker استفاده می‌کند
- اگر این خطا را دیدی، مطمئن شو نسخه v9.0.0 یا بالاتر اجرا می‌کنی

## نکات امنیتی

- **API Token**: توکن API کلادفلر فقط محلیاً برای ارتباط با Cloudflare API استفاده می‌شود و هرگز به هیچ سرور شخص ثالثی ارسال نمی‌شود.
- **UUID**: UUID فقط برای تولید کانفیگ VLESS استفاده می‌شود و در Worker دیپلوی‌شده شما جاسازی می‌شود.
- **سرور محلی**: سرور HTTP فقط به `127.0.0.1` متصل می‌شود — از سایر ماشین‌ها قابل دسترسی نیست.
- **بدون تلمتری**: برنامه هیچ داده‌ای به سرورهای خارجی ارسال نمی‌کند، جز فراخوان‌های Cloudflare API که خودتان آغاز می‌کنید.
