تمرکز نسخه v9

کمتر تنظیم، خروجی بهتر

مسیر پیشنهادی برای تازه‌کارها: Deploy کن، لینک Subscription را Paste کن، بعد روی «شروع اجرا» بزن. حالت هوشمند اول کانفیگ‌های اصلی BPB را تست می‌کند و اگر لازم شد Clean IPهای آماده را هم وارد تست می‌کند.

Atomic Mail / Email

۱) ایمیل پروژه را آماده کن

  1. اگر تیم بهت Atomic Mail داده، همان ایمیل را استفاده کن.
  2. اگر ایمیل نداری، روی دکمه زیر بزن و ایمیل قابل‌دسترسی بساز.
  3. همان ایمیل را برای Cloudflare وارد کن و رمز را نگه دار.
Cloudflare

۲) ساخت حساب Cloudflare

  1. روی «ثبت‌نام Cloudflare» بزن.
  2. Email و Password را وارد کن.
  3. وقتی وارد Dashboard شدی، همین نرم‌افزار را ادامه بده.
  4. اگر صفحه انتخاب Account دیدی، Account خودت/تیمت را انتخاب کن.
Deploy Assistant داخلی

۳) نصب Worker داخلی BPB

این بخش فایل Worker داخل پروژه را با Cloudflare API روی اکانت خودت Deploy می‌کند. اگر پنل/Worker آماده داری، مستقیم برو مرحله Subscription.

خطای رایج:

اگر مسیر خطا شامل accounts/cfat_... شد، یعنی Token را اشتباهی در فیلد Account ID گذاشتی. اول دکمه «تست Token و پیدا کردن Account ID» را بزن.

اول API Token را Paste کن، بعد «تست Token» را بزن. اگر Account ID پیدا شد، Deploy را بزن.
دقیقاً کجا کلیک کند؟

ساخت Token در Cloudflare

  1. دکمه API Tokens را بزن.
  2. در Cloudflare برو: My Profile → API Tokens.
  3. Create Token را بزن.
  4. اگر قالب آماده مناسب دیدی استفاده کن؛ وگرنه Create Custom Token.
  5. Permission: Account → Workers Scripts → Edit.
  6. Account Resources: همان Account خودت/تیمت.
  7. Continue to summary و بعد Create Token.
  8. Token را کپی کن و داخل نرم‌افزار Paste کن.
Subscription بعد از Deploy:

اگر نرم‌افزار لینک داد، همان را کپی کن. اگر لینک نداد: Cloudflare → Workers & Pages → bpb-panel → Visit، سپس مسیر /sub را باز کن.

IP Quality Scanner

۴) اسکن اختیاری، ولی دقیق‌تر

کاربر تازه‌کار لازم نیست زیاد درگیر اسکن شود. اگر کانفیگ اصلی BPB جواب نداد، این بخش چند endpoint خوب پیدا می‌کند و به مرحله تست منتقل می‌کند.

پورت‌های تست کیفیت:
نتیجه اسکن
اگر Subscription اصلی جواب نداد، این اسکن را اجرا کن. خروجی سالم در output/clean_ips.txt ذخیره می‌شود.
Subscription

۵) لینک را Paste کن

لینک Subscription را وارد کن و «چک کردن لینک» را بزن.
Smart Tester

۶) تولید و تست خروجی

پورت‌های تولید کانفیگ:
تنظیمات تست
برای کاربر تازه‌کار
  1. لینک Subscription را Paste کن.
  2. حالت هوشمند را دست نزن.
  3. روی دکمه شروع اجرا بزن.
  4. کانفیگ خروجی را از مرحله آخر کپی کن.
خروجی نهایی

۷) کانفیگ فعال پیشنهادی

بهترین کانفیگ در output/best_active_config.txt ذخیره می‌شود. همه کانفیگ‌های OK هم در output/working_configs.txt می‌آیند.
